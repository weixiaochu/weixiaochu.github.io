场景一：局域网内端口转发

方式一：netsh

服务器A处于局域网环境内，IP为192.168.1.200（同时作为虚拟机B的网关192.168.137.1），服务器A中安装有不同网段的虚拟机B，IP为192.168.137.2。A可实现对B的远程，但A同网段的其他机器无法ping通B

在服务器A执行如下CMD命令，建立端口转发规则192.168.1.200:13389 ==> 192.168.137.2:3389

#建立端口转发规则
netsh interface portproxy add v4tov4 listenaddress=192.168.1.200 listenport=13389 connectaddress=192.168.137.2 connectport=3389
#查看本机添加的所有规则
netsh interface portproxy show all
netstat -ano | findstr :3389
#删除指定规则
netsh interface portproxy delete v4tov4 listenaddress=192.168.1.200 listenport=13389

3、windows安装openssh ，安装教程

xxxxxxxxxx

场景三：本地端口转发

假设您处于一个不允许连接到特定服务器的专用网络上。比方说，你在工作，google被封锁。为了绕过这个，我们可以通过一台服务器创建一个不在我们的工作网络的隧道，从而可以访问google。

xxxxxxxxxx
ssh -L 9000:google.com:80 user@example.com

这里的关键是-L选项， 告诉我们正在进行本地端口转发，将端口9000转发到google.com:80， 80是HTTP的默认端口。现在，只需要打开浏览器然后访问：

xxxxxxxxxx
http://localhost:9000

SSH隧道的好处在于它们是加密的。没有人会看到你访问的网站，他们只看到SSH连接到你的服务器。

场景四：将本地应用暴露在公网

其实我要做的是将内网部署的web服务，通过外网ip能够访问，VPS你懂的，又贵，所以不能完全将一个很大的东西放在VPS上...

  • 登录公网主机，配置SSH

    xxxxxxxxxx
    vim /etc/ssh/sshd_config
    # ssh默认不允许远程主机转发端口
    GatewayPorts yes
    # 保持心跳，服务端每个60秒发送心跳给客户端
    ClientAliveInterval 60
    # 服务端发出的请求客户端未响应1次时断开连接
    ClientAliveCountMax 1

  • 在客户机（内网机）执行远程端口转发

    xxxxxxxxxx
    # 返回tty
    ssh -N -R 远程端口:localhost:本地端口 root@外网IP
    # 不返回tty
    ssh -nNT -R 远程端口:localhost:本地端口 root@外网IP
